Microservices are used to build the large application into small independent, scalable application.
Through this each application can be build in different technologies, can be deployed separately as per new development. 
It allow small teams to own specific services, deploy updates independently, and scale only what's needed, boosting productivity and responsiveness to market changes. 

Implement the "database per Microservices" pattern where each service has its own isolated data store.

Communication Between services

use REST API for synchronous call using httpClient

Service Discovery
------------------
Service discovery in microservices is the automatic process allowing services to find and communicate with each other dynamically, without hardcoded locations, 
using a central Service Registry (like Eureka or Consul) that tracks each service's network address (IP/port), enabling flexible scaling and resilient communication
in a constantly changing distributed system. Services register themselves with the registry, and clients query it to find available instances, with the registry 
often using health checks and load balancing to ensure reliable connections.
